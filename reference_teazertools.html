teazertools.submitter¶

This module provides the infrastructure to send job arrays of C++QED trajectory ensembles to the teazer cluster.

class teazertools.submitter.GenericSubmitter(argv=None)¶

This class generates various JobArray objects from a configuration file. For the syntax and usage, see Submitter.

act()¶

Submit all job arrays to the hpc cluster.

Parameters:
  • testrun (bool) – Perform a test run with only two seeds and T=1.
  • dryrun (bool) – Don’t submit anything, instead print what would be run on the nodes.
class teazertools.submitter.JobArray(script, basename=None, parSet={}, varPars=None, parameters={}, basedir='.', tempdir='/tmp', seeds=[1001], config={})¶

This class represents a job array to simulate a trajectory ensemble. A job array is characterized by a set of seeds, one seed for each trajectory, and a set of C++QED parameters identical for all seed. Functions are provided to submit the job array to the teazer cluster (test run is possible) or to simulate a specific seed locally. For submitted job arrays, a further job depending on the array can be submitted to calculate the ensemble averages of expectation values.

Parameters:
  • script (str) – The name of the executable to call. This has to be an absolute path if it is not in the systems $PATH.
  • basename (str) – The basename for output files. Defaults to the scriptname (without directory part).
  • parameters (dict) – The C++QED parameters.
  • basedir (str) – The directory for the output directory structure.
  • tempdir (str) – A temporary directory for C++QED output (submitted jobs write to local disk, the files are moved at the end)
  • seeds (list) – A list of seeds which defines the trajectory ensemble.
  • config (dict) – A dictionary to control various aspects of this class.

The possible values in config are:

Parameters:
  • averageids (dict) – A dictionary to define which columns of the output files contain expectation values, variances and standard deviations.
  • qsub (dict) – A dictionary of parameters passed to qsub. Each key:value pair corresponds to a -key value commandline option to qsub.
  • qsub_traj – Same as qsub, but items here are only applied to qsub submissions of trajectory job arrays.
  • qsub_average (dict) – Same as qsub, but items here are only applied to qsub submissions of averaging jobs.
  • qsub_test – Same as qsub, but items here are only applied to qsub submission if it is a testrun.
  • diagnostics – If True, print diagnostic messages to log files.
  • matlab – If True, convert trajectories and state vector files to matlab format.
  • average – If True, submit a job which calculates ensemble averages.
  • usetemp – Write data to temporary directory first (default True).
  • compress – Compress files (default True)
  • resume – Resume trajectories (default False)
  • testrun_t – Final time to integrate in testruns (default 1)
  • testrn_dt – -Dt for testruns (default None)
  • cluster – Each job should calculate this many trajectories (default 1)
diagnostics_after()¶

This function is called after the executable finished. It can be overloaded in subclasses. The default implementation writes a message to the log files.

diagnostics_before()¶

This function is called before the executable is called. It can be overloaded in subclasses. The default implementation writes a message with the hostname to the log file.

Parameters:

  • dryrun – If True, don’t simulate anything, but print a log message which contains the command that would have been run.

teazertools.helpers¶

This module defines some helper functions and the class VariableParameters.

class teazertools.helpers.VariableParameters(parameterValues={}, parameterGroups=(), combine=True)¶

This class represents a group of parameters to iterate over. It is one of the central classes of the teazertools.submitter, as it allows to specify parameter sweeps and allows slicing (e.g. keeping one parameter constant while sweeping the other).

Parameters can be grouped together with parameterGroups. Within each group of parameters, the values are iterated in parallel. If not all parameters within one group have the same amount of values, ValueError is raised. Between different groups of parameters, all possible combinations are generated. By default, each parameter is in a group of its own.

Example:

>>> import teazertools.helpers as helpers
>>> varPars=helpers.VariableParameters(parameterValues=dict(par1=[1,2],par2=[3,4],par3=[5,6]),parameterGroups=[['par1','par2']])
>>> list(varPars.parGen())
[{'par1': 1, 'par2': 3, 'par3': 5},
{'par1': 1, 'par2': 3, 'par3': 6},
{'par1': 2, 'par2': 4, 'par3': 5},
{'par1': 2, 'par2': 4, 'par3': 6}]
Parameters:
  • parameterValues (dict) – Dictionary of the structure {par1:[val1,val2,...], par2:[val1,val2,...], ...}
  • parameterGroups – List of the structure [[par1,par2,...],[par3,par4,...],...]
  • combine – Convenience parameter. If set to False, all parameters are put into one single group automatically.

:type combine:bool

parGen(subset=None)¶

Iterator over all parameter sets. Slicing is possible with the subset parameter.

Example:

>>> import teazertools.helpers as helpers
>>> varPars=helpers.VariableParameters(parameterValues=dict(par1=[1,2,3,4],par2=[-1,-2,-3,-4],par3=[5,6]),parameterGroups=[['par1','par2']])
>>> list(varPars.parGen())
[{'par1': 1, 'par2': -1, 'par3': 5},
{'par1': 1, 'par2': -1, 'par3': 6},
{'par1': 2, 'par2': -2, 'par3': 5},
{'par1': 2, 'par2': -2, 'par3': 6},
{'par1': 3, 'par2': -3, 'par3': 5},
{'par1': 3, 'par2': -3, 'par3': 6},
{'par1': 4, 'par2': -4, 'par3': 5},
{'par1': 4, 'par2': -4, 'par3': 6}]
>>> list(varPars.parGen(subset=dict(par1=1)))
[{'par1': 1, 'par2': -1, 'par3': 5}, {'par1': 1, 'par2': -1, 'par3': 6}]
>>> list(varPars.parGen(subset=dict(par1=[2,3],par3=5)))
[{'par1': 2, 'par2': -2, 'par3': 5}, {'par1': 3, 'par2': -3, 'par3': 5}]
Parameters:subset (dict) – A subset to which the list of parameter sets is restrained.
Returns:Iterator over parameter sets (dictionaries).
Retval :iterator

teazertools.mean¶

This module provides functions to average a C++QED MCWF trajectory and to save the results to a file. It is also possible to convert the results to a format readable by matlab. The main function is calculateMeans().

teazertools.mean.calculateMeans(basename, evslist=None, expvals=[], variances=[], varmeans=[], stdevs=[], stdevmeans=[], datadir='.', outputdir='.', matlab=True, bz2only=False)¶

Calculate the mean expectation values, mean variances and mean standard deviations from an ensemble of C++QED MCWF trajectories. The results are saved to a file.

Parameters:
  • basename (str) – String with the basename of the files to import, e.g. “RingCavity”
  • expvals (list) – List of indices which contain normal expectation values (Note: first column has index 1)
  • variances – List of indices which contain variances
  • varmeans – Corresponding mean values to the variances (if given, this list must have the same length as variances). This can be left empty, in which case the mean values are expected in the column just before each variance, respectively.
  • stdevs (list) – List of indices which contain standard deviations
  • stdevmeans (list) – See varmeans
  • datadir (str) – Directory name from which files are loaded.
  • outputdir (str) – Directory name where output is written to. If this is None, don’t write anny output.
  • matlab – Also convert results to matlab format and save a .mat file.
Returns:

An array containing the averaged expectation values, standard deviations and variances.
Return type:

np.ndarray
teazertools.mean.calculateRho(basename, dirname='.')¶

This function averages the density matrices in a C++QED MCWF trajectory ensemble. Note that the equivalent matlab code is found to be orders of magnitude faster because of multi-threading.

Parameters:
  • basename (str) – String with the basename of the files to import.
  • dirname (str) – Directory name from which files are imported.
Returns:

A tuple (timevec,rho), first entry is the a vector of times at which statevector were printed, the second entry is a np.ndarray where each row i corresponds to a averaged density matrix at time timevec[i].
Retval :

tuple
